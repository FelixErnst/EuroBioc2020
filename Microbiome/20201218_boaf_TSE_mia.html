require one additional data type to match phyloseq: sequencing data

Two options

  1. Create MicrobiomeExperiment providing the refSeq slot while inheriting from TSE

or

  1. Add refSeq slot to TSE

 

Currently this is implemented as 1., but the PR for adding it to TSE is also prepared.

Converting taxonomic tabular data to tree data

TreeSummarizedExperiment already contains functions for generating data from hierarchical tabular data. mia adds wrapper functions.

taxonomyTree(se)

se <- addTaxonomyTree(se)

Functions for microbiome workflows

Using the taxonomic levels

Data can be aggregated on a selected taxonomic rank.

x <- agglomerateByRank(se, "Phylum")
dim(se)
## [1] 19216    26
dim(x)
## [1] 67 26

Internally, constructs a factor to pass to mia::mergeRows, which uses scater/scuttle functions.

Functions for microbiome workflows

Using the taxonomic levels

mia::splitByRanks is a convenience wrapper around agglomerateByRank.

Results can be added to the alternative experiment slots inherited from SCE.

altExps(se) <- splitByRanks(se, ranks = taxonomyRanks(se))
altExpNames(se)
## [1] "Kingdom" "Phylum"  "Class"   "Order"   "Family"  "Genus"   "Species"

Analysis can be performed individually on each taxonomic level.

altExps(se) <-
   lapply(altExps(se),
          function(y){
              rowData(y)$prevalence <-
                  getPrevalence(y, detection = 1/100, sort = FALSE,
                                abund_values = "counts", as_relative = TRUE)
              y
          })

Plotting functions split of to miaViz package

top_phyla <- getTopTaxa(altExp(se,"Phylum"))
plotRowTree(x[rowData(x)$Phylum %in% top_phyla,], edge_colour_by = "Phylum",

  • in Bioc-devel the functions calculateMDS/runMDS from scater accept any distance measurement for so that all vegdist methods can be used for MDS.
